# HerdTrackr - Project Context for Claude

## Project Overview

HerdTrackr is a **React Native + Expo** mobile app for livestock management, targeting farmers in South Africa. It supports cattle, buffalo, sheep, goats, horses, pigs, and game animals.

### Key Technologies

- **Frontend**: React Native (0.81.4) + Expo (54)
- **Database**: WatermelonDB (local SQLite) + Supabase (cloud sync)
- **Auth**: Supabase Auth with magic links (no passwords)
- **Navigation**: React Navigation 7
- **Styling**: Custom themed system with ThemedStyle
- **State**: React Context for auth, database, and sync

### Architecture

- **Offline-first**: All data stored locally in SQLite, syncs when online
- **Multi-tenant**: Organizations with team members (admin/worker roles)
- **Role-based**: Admins can delete and invite, workers can add/edit only

## Code Patterns & Conventions

### 1. ThemedStyle Usage

**ALWAYS use function call syntax:**

```typescript
// ✅ Correct
<View style={themed($container)} />

// ❌ Wrong
<View style={themed.$container} />
```

**Define styles at bottom of file:**

```typescript
const $container: ThemedStyle<ViewStyle> = ({ spacing, colors }) => ({
  flex: 1,
  paddingHorizontal: spacing.lg,
  backgroundColor: colors.background,
})
```

### 2. Screen Structure

```typescript
import { AppStackScreenProps } from "@/navigators"

interface ScreenNameProps extends AppStackScreenProps<"ScreenName"> {}

export function ScreenNameScreen({ navigation, route }: ScreenNameProps) {
  const { themed } = useAppTheme()
  // component logic
}
```

### 3. Database Queries

**Always filter by organization:**

```typescript
const animals = await database
  .get<Animal>("animals")
  .query(
    Q.where("organization_id", currentOrg.id),
    Q.where("is_deleted", false),
    Q.sortBy("visual_tag", Q.asc)
  )
  .fetch()
```

**Use observables for reactive data:**

```typescript
const subscription = database
  .get<Animal>("animals")
  .query(Q.where("organization_id", currentOrg.id))
  .observe()
  .subscribe((data) => setAnimals(data))

return () => subscription.unsubscribe()
```

### 4. Create/Update Patterns

**Always wrap in database.write():**

```typescript
await database.write(async () => {
  return await database.get<Animal>("animals").create((animal) => {
    animal.organizationId = currentOrg.id
    animal.rfidTag = data.rfidTag
    animal.isDeleted = false
  })
})
```

### 5. Navigation

**Type-safe navigation:**

```typescript
navigation.navigate("AnimalDetail", { animalId: "123" })
```

**Replace instead of push for setup screens:**

```typescript
navigation.replace("Main")
```

## Common Tasks

### Adding a New Screen

1. Create `app/screens/[Name]/[Name].tsx`
2. Create `app/screens/[Name]/index.ts`
3. Add type to `app/navigators/navigationTypes.ts`
4. Add route to `app/navigators/AppNavigator.tsx`
5. Import and use in navigation

### Adding a New Model

1. Add migration in `app/db/schema.ts` (increment version)
2. Add table schema in same file
3. Create model in `app/db/models/[Name].ts`
4. Export in `app/db/models/index.ts`
5. Register in `app/db/index.ts`
6. Create Supabase migration in `supabase/migrations/`

### Adding a New Hook

1. Create `app/hooks/use[Name].ts`
2. Export data-fetching hook and actions hook separately:
   ```typescript
   export function use[Name]() { /* fetch data */ }
   export function use[Name]Actions() { /* mutations */ }
   ```

## Important Rules

### Database

- **Always check `currentOrg`** before queries
- **Always soft-delete** with `isDeleted = true`
- **Always use observables** for lists that auto-update
- **Always include organization_id** in create operations
- **Never delete records** permanently

### Styling

- **Use ThemedStyle** for all styles
- **Use spacing tokens**: `spacing.xs`, `spacing.sm`, `spacing.md`, `spacing.lg`, `spacing.xl`
- **Use color tokens**: `colors.text`, `colors.background`, `colors.palette.primary500`, etc.
- **Never use magic numbers** - use theme values

### Components

- **Prefer editing existing** files over creating new ones
- **Use existing components**: Screen, Text, Button, TextField, Icon, ListItem
- **Follow established patterns** - check similar screens first

### Navigation

- **Always type navigation** with AppStackScreenProps
- **Use replace()** for auth/setup flows (can't go back)
- **Use navigate()** for normal flows
- **Use goBack()** for dismissals

### Row-Level Security

All tables have RLS enabled. Users can only access data from organizations they're members of:

```sql
CREATE POLICY "Users can access their org data"
  ON public.[table_name] FOR ALL
  USING (
    EXISTS (
      SELECT 1 FROM public.memberships
      WHERE memberships.organization_id = [table_name].organization_id
        AND memberships.user_id = auth.uid()
        AND memberships.is_active = true
    )
  );
```

## User Roles

### Admin
- Can invite/remove users
- Can change user roles
- Can delete records permanently
- Can change organization settings

### Worker
- Can add/edit animals, health records, pastures
- Can view all data in their organization
- Cannot delete or invite

Check roles:
```typescript
const { members } = useTeam()
const currentMember = members.find(m => m.userId === user?.id)
const isAdmin = currentMember?.role === "admin"
```

## Common Gotchas

1. **ThemedStyle syntax**: Use `themed($style)` NOT `themed.$style`
2. **Database writes**: Always wrap in `database.write(async () => {})`
3. **Organization filter**: Always include `Q.where("organization_id", currentOrg.id)`
4. **Soft deletes**: Filter with `Q.where("is_deleted", false)`
5. **Observables**: Unsubscribe in useEffect cleanup
6. **Navigation types**: Must add to AppStackParamList
7. **Supabase RLS**: Use service role key for admin operations, publishable key for client
